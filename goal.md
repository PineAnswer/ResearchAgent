# 文献研究工作台桌面端视觉重构

## 目标

将现有原生 HTML、CSS、JavaScript 前端完整重构为 `Apple Pages x Nature`
风格的专业学术编辑工作台。新版采用冷白、墨色和松绿，强调成果阅读、
编辑式留白与克制交互，并用真实图标替代字符图标。

保留全部研究功能、后端 API、数据模型和研究阶段，不改变现有业务流程。

## 设计方向

- 桌面网页专用，最低支持 `1280x720`，Windows、Chrome 和 Edge 优先。
- 保留 `Evidence Research Agent / 文献研究工作台` 中英双语品牌。
- 使用舒展但不浪费空间的编辑式留白，核心成果和当前操作尽量首屏可见。
- 页面以最终研究成果为中心；未完成项目根据阶段显示当前最重要任务。
- 不使用渐变、装饰光斑、营销式 Hero、巨大空区或夸张动效。
- 不实现手机和平板布局、暗色模式、营销首页或后端功能变更。

## 视觉系统

- 建立统一的 4/8px 间距尺度和固定字号层级，所有字距设为 `0`。
- 控件高度统一为 32px 或 40px，卡片和面板圆角不超过 8px。
- 色板固定为冷白 `#F6F7F5`、纸白 `#FFFFFF`、墨色 `#191C1A`、
  中性灰和松绿 `#245B48`。
- 红色仅用于危险操作，警告色仅在必须区分状态时出现。
- 阴影只用于抽屉、菜单和浮层，不用阴影堆叠页面层级。
- 本地打包 Inter 作为界面字体，Noto Serif CJK SC 作为标题和研究成果字体，
  并保留对应开源许可证。
- 本地打包 Lucide 浏览器图标库，替换字符图标。
- 所有图标按钮使用稳定尺寸，并提供 tooltip、`aria-label` 和键盘焦点样式。

## 布局与导航

- 顶部改为 64px 高的编辑部刊头，品牌位于左侧，服务状态和工具菜单位于右侧。
- 项目侧栏在宽屏默认展开为 280px，在 `1280-1439px` 默认收窄为 72px。
- 用户可以手动展开或收窄侧栏，状态保存到
  `research-agent.sidebar-state`，不保存项目数据或敏感信息。
- 收窄侧栏保留新建研究、项目库和当前项目状态的图标入口。
- API 文档、按项目 ID 打开和其他低频工具移入“更多”菜单。
- 删除项目移入项目操作菜单，继续保留不可撤销的二次确认。

## 核心工作流

### 新建研究

- “发起新研究”在主工作区打开研究简报，不再把长表单塞进侧栏。
- 主题和研究问题作为主要字段，论文数量和检索轮次作为次级设置。
- 精简说明文字并统一术语，不改变字段含义或提交格式。

### 项目详情

- 已完成项目首先展示最终综述、摘要、章节、引用与参考文献。
- 未完成项目根据阶段显示检索进度、候选审核或继续研究操作。
- 将当前阶段和关键数量整理为安静的元数据条，减少指标卡片堆叠。
- 使用窄阅读列保证长篇中文综述的行宽、段落间距和引用可读性。

### 候选论文审核

- 将论文卡片改为编辑式列表，统一勾选区、标题、作者、来源、年份和摘要层级。
- 筛选工具栏保持可见，候选数量与选中数量使用紧凑的内联统计。
- 将补充检索和反馈参数组织成一个连贯表单，不使用卡片嵌套。
- 保留全部保留、全部取消、更新候选集、确认选择和证据不足结束项目功能。

### 成果与高级信息

- 主区仅保留成果内容和当前任务，研究过程与原始产物移入右侧检查器。
- 检查器宽度为 420px，提供“过程 / 产物”标签和“可读 / 原始”切换。
- 检查器支持关闭按钮、`Esc` 关闭、键盘焦点管理和轻微过渡动画。
- 现有所有产物渲染器、原始 JSON 和时间线数据继续可访问。

## 实现约束

- 主要修改集中在 `src/research_agent/api/frontend/` 下的 HTML、CSS 和 JavaScript。
- 后端接口、请求结构、数据库和研究 Agent 行为保持不变。
- 静态资源新增 Lucide 浏览器包、Inter、Noto Serif CJK SC 及许可证文件。
- DOM 结构和 JavaScript 渲染逻辑必须同步更新，不留下失效选择器或事件监听器。
- 动效只用于侧栏、抽屉、菜单、按钮反馈和状态切换。
- 完整支持 `prefers-reduced-motion`，禁用后不影响任何操作。

## 测试计划

- 保持现有 Python 测试全部通过，并补充字体、图标和静态资源可访问性测试。
- 对 `app.js` 执行 JavaScript 语法检查。
- 扫描前端，确保按钮和装饰区域不再使用 emoji 或字符图标。
- 使用独立临时端口和临时数据目录启动测试服务，不接触用户现有研究数据库。
- 使用 Playwright 覆盖空状态、新建、运行中、候选审核、完成成果、检查器、
  侧栏展开和侧栏收窄状态。
- 截图验收尺寸为 `1280x720`、`1440x900` 和 `1920x1080`。
- 检查文字溢出、组件重叠、横向滚动、间距一致性、首屏层级和长文阅读效果。
- 在 Chrome 和 Edge 完成交互冒烟测试，包括键盘导航、焦点、tooltip、菜单、
  抽屉和删除确认。
- 临时测试服务在视觉验收完成后立即关闭。

## 验收标准

- 所有现有研究功能和 API 行为保持可用。
- 全部前端状态使用同一套颜色、字体、图标、间距和控件规范。
- 页面中不存在用于操作或装饰的 emoji、字符图标、渐变和装饰性光斑。
- `1280x720` 下核心任务可操作，`1440x900` 和 `1920x1080` 下布局舒展但不松散。
- 最终成果是项目页的视觉中心，流程和原始数据不会干扰阅读。
- 无文本截断、无控件重叠、无意外横向滚动、无卡片嵌套。
- 自动化测试、JavaScript 检查与 Playwright 视觉验收全部通过。

## 发布假设

- 本任务只完成前端设计与实现，不修改后端能力。
- 实现完成后不自动提交或推送 GitHub，除非用户另行要求。
