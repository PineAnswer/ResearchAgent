# 综述研究工作台实现说明

## 目标

综述生成后，在正文右侧提供一个可长期使用的研究工作台，支持范围受控的问答、选段批注、普通笔记、问答保存和研究收藏。窄屏下工作台自动移动到正文下方。

## 多轮聊天

右侧聊天框直接调用聊天模型，不设置回答范围、引用格式或材料充分性限制。系统会自动把当前综述、用户选中的文字和项目入选论文作为可选上下文，同时允许模型使用自身知识正常回答。

接口为 `POST /api/projects/{project_id}/assistant/stream`，响应格式为 SSE：

- `delta`：模型新生成的文本片段；
- `done`：本轮生成完成；
- `error`：模型调用失败及原因。

浏览器会逐段渲染流式 Markdown，并把最近 40 条用户与助手消息随下一轮请求发送，从而支持连续追问。模型调用失败时直接显示错误，不再返回原文摘录。单次选段限制为 12,000 字，问题限制为 4,000 字，附加研究上下文最多 120,000 字，这些限制仅用于控制请求体大小。

## 批注与笔记

研究级记录存储在 SQLite 的 `research_notes` 表中，并按当前用户和项目隔离：

- `note`：未选择原文时保存的普通笔记；
- `annotation`：保存选中的综述原文及用户批注；
- `qa`：保存问题、回答、范围内选段和引用快照。

对应接口：

- `GET /api/projects/{project_id}/notes`
- `POST /api/projects/{project_id}/notes`
- `PATCH /api/projects/{project_id}/notes/{note_id}`
- `DELETE /api/projects/{project_id}/notes/{note_id}`

删除项目时，数据库外键级联删除该项目的研究记录。批注必须同时包含选中文本和批注内容；保存问答必须同时包含问题和回答。

## 收藏与恢复

研究项目及产物原本就会持久化。工作台中的“收藏研究”复用会话的 `pinned/pinned_at` 字段：收藏后研究在左侧历史列表置顶并显示收藏状态，取消收藏不会删除项目、产物、问答或笔记。重新打开项目时，最终综述与研究级记录分别从项目快照和研究笔记接口恢复。
