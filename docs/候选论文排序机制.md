# 候选论文排序机制

本文说明检索完成后、候选论文提交给用户筛选前的统一排序机制。实现位于
`src/research_agent/application/candidate_ranking.py`。

## 设计目标

1. 研究问题相关性始终是主要因素。
2. 单篇论文影响程度使用同领域、同年份、同文献类型的归一化指标。
3. 论文只被一个渠道检索到时，不因渠道缺失被扣影响分；渠道覆盖影响数据置信度。
4. CCF、JCR 等评价只衡量发表场馆，不能代替单篇论文影响程度。
5. 避免列表前部被内容高度相似的论文占满。
6. 所有缺失值、异常值、极端值和撤稿状态都采用显式兜底。

## 总分

候选论文综合分满分 100：

```text
综合分 = 55% × 相关性
       + 30% × 论文影响程度
       + 10% × 场馆权威性
       +  5% × 候选差异性
```

某个分项缺失时，从分母中移除该分项并对剩余权重重新归一化。缺少 JCR、CCF、
Semantic Scholar 或 Crossref 数据不会自动得到零分。

Agent 已给出三态筛选结果时，先按 `include → uncertain → exclude` 分组，再在组内按综合分排序。
撤稿论文保留实际影响程度，但综合分上限设为 5，并显示风险标签。

## 相关性（55%）

每条查询分别计算查询词在标题和摘要中的覆盖：

```text
单查询相关性 = 70% × 标题词覆盖率 + 30% × 摘要词覆盖率
总体相关性 = 70% × 最佳单查询相关性 + 30% × 全部查询平均相关性
```

这样可以兼顾最直接的命中和多个互补查询的覆盖。相关性范围为 0–100。空查询时复用已有的
有限相关性分；非数值、NaN、无穷大和负值均被安全裁剪。

## 论文影响程度（30%）

```text
影响程度 = (65% × P + 20% × Q + 15% × M) / 可用分项权重
```

- `P`：领域归一化引用百分位。
- `Q`：Semantic Scholar 高价值引用数在比较集合中的百分位。
- `M`：最近两个完整年度的年均新增引用在比较集合中的百分位。

### 多渠道融合

领域归一化引用百分位：

```text
P = (50% × OpenAlex
   + 35% × Semantic Scholar
   + 15% × Crossref) / 可用渠道权重
```

近期增长百分位：

```text
M = (60% × OpenAlex
   + 30% × Semantic Scholar
   + 10% × Crossref) / 可用渠道权重
```

OpenAlex 原生的 `citation_normalized_percentile` 优先使用。Semantic Scholar 和 Crossref
只有原始引用数时，系统在同领域、同年份、同类型候选集合内计算中位秩百分位。比较集合过小时
依次放宽为相邻年份、父级范围和全部同类型候选。最终只剩一个数值时，使用保守的饱和函数
`1-exp(-citations/20)`，避免单条记录被误判成第 0 或第 100 百分位。

原始引用数不会跨渠道相加。预印本和正式版本应通过 DOI 或规范化标题合并。

### 影响数据置信度

置信度不进入影响分：

```text
影响置信度 = 55% × 渠道覆盖
           + 25% × 影响分项覆盖
           + 20% × 渠道一致性
```

只有 OpenAlex 数据的高影响论文仍可获得高影响分，同时显示较低置信度。渠道百分位差超过
50 个百分点时，一致性降至 0。

## 场馆权威性（10%）

当前映射：

| 评级 | 分数 |
| --- | ---: |
| CCF-A / B / C | 100 / 75 / 55 |
| JCR Q1 / Q2 / Q3 / Q4 | 90 / 70 / 50 / 30 |
| Nature Portfolio | 95 |

同一场馆命中多个体系时采用最高分，同时保留全部标签和评级来源。没有适用评级时该分项缺失，
不会记为 0。影响因子没有直接进入公式，因为不同学科的影响因子不可横向比较。

## 候选差异性（5%）

系统使用贪心 MMR 风格重排。第一篇差异性为 100；后续论文根据标题、摘要、场馆和学科词集合
与已排序论文计算最大 Jaccard 相似度：

```text
差异性 = 100 × (1 - 与已选论文的最大相似度)
```

空文本采用中性值 50。差异性仅占 5%，用于打破高度相似候选之间的排序，不会显著压过相关性。

## 边界与异常兜底

- 空候选集：直接返回空列表。
- 空查询：复用已有相关性分，缺失时记为 0。
- 缺少影响数据：`impact_score=null`，综合分对其他可用分项重加权。
- 单渠道论文：按该渠道指标计算影响分，降低置信度，不降低影响分。
- 新论文：引用为零可以得到低影响百分位，但前端应提示“影响仍在形成”。
- 负数、NaN、无穷大：视为缺失或裁剪到合法范围。
- 极大引用数：先转成百分位，避免少数极端论文支配总分。
- 年份或学科缺失：逐级放宽比较集合；仍不足时启用保守饱和函数。
- 多轮检索重复论文：合并来源、查询、引用指标、学科和较长摘要，再全局重排。
- 撤稿：保留影响分供研究史判断，综合分最多为 5，并显示撤稿风险。
- 排序同分：依次按相关性、影响程度、年份、标题稳定排序，保证分页顺序可复现。

## 前端呈现

每张候选卡展示：

```text
综合 87  影响 93  相关 84  场馆 100
```

鼠标悬停“影响”可查看引用百分位组成、缺失渠道说明和数据置信度；悬停“综合”可查看全部
分项。论文卡仍必须提供一句“筛选依据 / 核心内容”，供用户进行最终人工筛选。
